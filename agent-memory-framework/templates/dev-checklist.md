# 开发检查清单

开发任何新功能时，按以下顺序执行：

1. **定义测试用例** — 先明确"什么算做完了"
   - [ ] 在测试文件中添加该功能的验证场景

2. **实现功能** — 开发代码
   - [ ] 后端逻辑实现
   - [ ] 错误处理

3. **集成** — 确保新功能接入主系统
   - [ ] [列出你的项目中需要更新的集成点]
   - [ ] 例: 路由注册、配置更新、依赖注入等

4. **前端适配** — 如有 UI
   - [ ] 对应的前端展示
   - [ ] 状态管理

5. **测试** — 运行验证
   - [ ] 单元测试通过
   - [ ] 集成测试通过
   - [ ] 手动验证

6. **提交** — Git
   - [ ] 有意义的 commit message
   - [ ] push 到远程

> ⚠️ 例: 写完后端 API 后，别忘了更新前端对应的调用
